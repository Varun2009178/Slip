# Gmail v2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace mock data + AI with real Gmail (browser OAuth), and add a warm-paper colorway.

**Architecture:** `src/lib/gmail.ts` owns auth (GIS token popup) and all Gmail REST calls, with pure, unit-tested helpers for parsing inbound messages and building outbound MIME. `Email` type moves to `src/lib/types.ts` and mirrors Gmail semantics (`unread`, `starred`, `threadId`, `rfcMessageId`). App gains a connect screen and async actions; store/read/done and mock data are deleted. Theme via `data-theme` attribute + CSS variables.

**Tech Stack:** unchanged (Vite/React/TS/Vitest) minus `@anthropic-ai/sdk`, plus Google Identity Services script loaded at runtime.

Spec: `docs/superpowers/specs/2026-07-01-tiny-email-client-design.md` (v2 section)
